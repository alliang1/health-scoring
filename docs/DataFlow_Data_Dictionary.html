DataFlow Data Dictionary

Nexus Cloud enterprise CRM platform

This data dictionary documents the entities in Nexus Cloud's DataFlow CRM system. It uses a relational structure with three entities to track customer accounts, contacts, and sales pipeline.

Overview / Entities

accounts

Customer and prospect records. Each account represents a company-level relationship.

Owner: Sales
33
Total Records
11
Attributes
Attributes
Attribute Type Required Description
idPK Text Yes Unique identifier (e.g., "df-a1")
name Text Yes Company name
parent_idFK Text Optional Links subsidiary to parent account. Null for top-level accounts.
industry Text Yes Industry classification
annual_revenue Currency Yes Company annual revenue in USD
employees Number Yes Employee count
address Text Yes Primary address
status Picklist Yes Account status Values: customer prospect
contract_value Currency Optional Annual contract value (ARR) in USD. Null for prospects.
contract_start Date Optional Contract start date (YYYY-MM-DD). Null for prospects.
contract_end Date Optional Contract end date (YYYY-MM-DD). Null for prospects.
Relationships
parent_id ← accounts.id self-referential (parent-child)
id → contacts.account_id one-to-many
id → opportunities.account_id one-to-many
Sample Records
id name parent_id industry annual_revenue employees status contract_value

df-a4 Coastal Logistics NULL Transportation $75,000,000 300 prospect NULL
df-a5 Dynamo Tech NULL Technology $30,000,000 200 customer $350,000
Notes

Accounts have status "customer" or "prospect". Prospects have no contract fields (contract_value, contract_start, contract_end are all null). The contract_value represents annual recurring revenue (ARR) for that specific account. Parent/subsidiary relationships are tracked via parent_id.

Overview / Entities

opportunities

Sales pipeline tracking. Each opportunity represents a potential or completed deal linked to one account.

Owner: Sales
40
Total Records
7
Attributes
Attributes
Attribute Type Required Description
idPK Text Yes Unique identifier (e.g., "df-o1")
account_idFK Text Yes Reference to the account this opportunity belongs to
name Text Yes Opportunity name
amount Currency Yes Deal value in USD
stage Picklist Yes Sales pipeline stage Values: discovery proposal negotiation closed_won closed_lost
close_date Date Yes Expected or actual close date (YYYY-MM-DD)
product Picklist Yes Product offering Values: DataFlow Enterprise DataFlow Standard DataFlow Analytics
Relationships
account_id ← accounts.id many-to-one
Sample Records
id account_id name amount stage close_date product
df-o1 df-a1 Apex Enterprise Expansion $450,000 closed_won 2024-01-10 DataFlow Enterprise
df-o2 df-a1 Apex Analytics Add-on $120,000 negotiation 2026-03-15 DataFlow Analytics

df-o5 df-a4 Coastal Logistics Discovery $275,000 discovery 2026-06-30 DataFlow Standard
Notes

Stage values track deals through the sales pipeline. closed_won and closed_lost are terminal stages. The amount field represents the total deal value, not annual recurring revenue.
